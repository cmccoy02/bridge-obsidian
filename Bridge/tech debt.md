# Tech Debt

Updated: 2026-02-24  
Context: [[Index-Bridge]]

## Core Thesis
Technical debt is no longer just "messy code." In AI-accelerated teams, debt is now a portfolio of liabilities across code, architecture, data, tests, security, and documentation. Teams that win treat debt as a managed financial system with explicit underwriting, interest tracking, and repayment SLAs.

## Operating Standard (Bridge)
1. Every meaningful debt item gets a TDR (technical debt record).
2. Every TDR has an owner, interest estimate, and next review date.
3. Debt without measured interest is not prioritized above feature work.
4. AI-generated changes require stronger verification gates than human-only diffs.
5. Architectural and documentation debt are first-class, not "later" categories.
